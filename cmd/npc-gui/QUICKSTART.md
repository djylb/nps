# NPC GUI 快速开始指南

欢迎使用 NPC GUI - 一个易用的 NPS 客户端图形界面管理工具！

## 什么是 NPC GUI？

NPC GUI 是一个跨平台的桌面应用程序，让您可以通过友好的图形界面管理 NPC（NPS 客户端）配置，无需记忆复杂的命令行参数。

## 主要功能

✅ **多配置管理** - 保存和管理多个服务器配置  
✅ **一键启停** - 点击按钮即可启动或停止客户端  
✅ **实时日志** - 查看客户端运行状态和日志信息  
✅ **支持所有连接类型** - TCP、TLS、KCP、QUIC、WebSocket、WebSocket TLS  
✅ **完整配置选项** - 代理、DNS、Keep-Alive 等高级选项  

## 快速开始

### 步骤 1: 下载安装

从 [Releases 页面](https://github.com/Yourdaylight/jqhl/releases) 下载适合您系统的版本：

- **Windows**: 下载 `npc-gui-windows-amd64.exe`
- **Linux**: 下载 `npc-gui-linux-amd64`
- **macOS**: 下载 `npc-gui-darwin-universal`

### 步骤 2: 运行程序

**Windows**:
- 双击 `npc-gui-windows-amd64.exe` 运行
- 如果出现安全警告，点击"仍要运行"

**Linux**:
```bash
chmod +x npc-gui-linux-amd64
./npc-gui-linux-amd64
```

**macOS**:
```bash
chmod +x npc-gui-darwin-universal
./npc-gui-darwin-universal
```

### 步骤 3: 添加配置

1. 点击界面中的 **"+ Add New"** 按钮
2. 填写配置信息：
   - **配置名称**: 给配置起个容易识别的名字（如"公司服务器"）
   - **服务器地址**: NPS 服务器地址，格式为 `域名:端口` 或 `IP:端口`
     - 例如：`example.com:8024` 或 `192.168.1.100:8024`
   - **验证密钥**: 从 NPS 服务器管理页面获取的客户端 vkey
   - **连接类型**: 选择连接方式（推荐 TLS 或 TCP）
3. 点击 **"Save Configuration"** 保存

### 步骤 4: 启动客户端

1. 在配置列表中找到刚才添加的配置
2. 点击配置右侧的 **"Start"** 按钮
3. 状态会变为 **"● Running"** 表示客户端正在运行
4. 切换到 **"Logs"** 标签页查看连接日志

### 步骤 5: 停止客户端

- 点击配置右侧的 **"Stop"** 按钮即可停止客户端

## 配置说明

### 基本配置

| 配置项 | 说明 | 示例 |
|--------|------|------|
| 配置名称 | 用于识别配置的名称 | "办公室服务器" |
| 服务器地址 | NPS 服务器地址和端口 | `nps.example.com:8024` |
| 验证密钥 | 客户端唯一标识符 | `abc123xyz456` |
| 连接类型 | 与服务器的连接方式 | TCP / TLS / KCP / QUIC / WS / WSS |

### 高级配置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 代理 URL | 使用 SOCKS5 代理连接 | 留空 |
| DNS 服务器 | 自定义 DNS 服务器 | 8.8.8.8 |
| Keep Alive | 保持连接心跳间隔（秒） | 0（默认） |
| 自动重连 | 断线后自动重新连接 | ✅ 启用 |
| 跳过 TLS 验证 | 不验证服务器证书 | ❌ 禁用 |
| 禁用 P2P | 不使用 P2P 连接 | ❌ 禁用 |

### 日志级别

选择日志详细程度：
- **Trace**: 最详细，用于调试
- **Debug**: 调试信息
- **Info**: 常规信息（推荐）
- **Warn**: 警告信息
- **Error**: 仅错误信息

## 常见问题

### Q: 客户端启动后无法连接？

**A**: 请检查：
1. 服务器地址是否正确（包括端口号）
2. 验证密钥是否正确
3. 服务器是否在运行
4. 防火墙是否允许连接
5. 在 "Logs" 标签查看详细错误信息

### Q: 如何知道客户端是否成功连接？

**A**: 
1. 查看配置状态是否显示 "● Running"（绿色）
2. 切换到 "Logs" 标签查看日志
3. 成功连接会显示类似 "Client connected successfully" 的消息

### Q: 可以同时运行多个客户端吗？

**A**: 可以！您可以创建多个配置，并同时启动它们连接到不同的服务器。

### Q: 配置保存在哪里？

**A**: 配置文件保存在：
- Windows: `C:\Users\你的用户名\.npc-gui\configs.json`
- Linux: `~/.npc-gui/configs.json`
- macOS: `~/.npc-gui/configs.json`

### Q: 如何备份配置？

**A**: 复制上述配置文件即可备份所有配置。

### Q: 忘记了服务器的验证密钥？

**A**: 需要登录 NPS 服务器的 Web 管理界面，在"客户端"页面查看或生成新的 vkey。

### Q: TLS 和 TCP 连接有什么区别？

**A**: 
- **TCP**: 普通连接，速度快但不加密
- **TLS**: 加密连接，更安全但略慢（推荐）
- **KCP**: 基于 UDP，适合高延迟网络
- **QUIC**: 新协议，结合了 TCP 和 UDP 的优点
- **WS/WSS**: WebSocket 连接，可以穿透某些防火墙

### Q: "Skip TLS Verification" 什么时候使用？

**A**: 仅在以下情况使用：
- 服务器使用自签名证书
- 开发测试环境
- **注意**: 生产环境不建议启用，会降低安全性

## 使用技巧

### 💡 技巧 1: 为配置起有意义的名称

使用描述性名称，如：
- ✅ "公司办公网络 - TLS"
- ✅ "家用服务器 - TCP"
- ❌ "配置1"
- ❌ "test"

### 💡 技巧 2: 生产环境使用 TLS

TLS 连接提供加密传输，保护您的数据安全。

### 💡 技巧 3: 启用自动重连

默认启用自动重连，这样网络临时中断后会自动恢复连接。

### 💡 技巧 4: 定期查看日志

在 "Logs" 标签页可以查看客户端运行状态，及时发现问题。

### 💡 技巧 5: 多配置管理

如果您有多个 NPS 服务器，可以为每个服务器创建一个配置，方便切换。
